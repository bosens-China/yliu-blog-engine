# 指南：如何写作

本博客引擎的核心设计，就是将 GitHub Issues 作为您的内容管理系统 (CMS)。这种方式让您可以专注于写作，而无需关心复杂的后台。

## 基础操作

### 创建一篇文章

在您作为内容源的 GitHub 仓库中，点击 **`New issue`** 即可开始撰写一篇新文章。

- **文章标题**: Issue 的标题 (`title`) 会直接成为您文章的标题。
- **文章内容**: Issue 的正文 (`body`) 就是文章的主体内容。我们支持所有标准的 **GitHub Flavored Markdown** 语法，包括代码块、表格、引用、图片等。

### 发布与隐藏文章

文章的发布状态与 Issue 的状态直接关联：

- **发布**: 保持 Issue 为 **`Open`** 状态，文章就会出现在您的博客上。
- **隐藏 (草稿)**: 将 Issue **`Close`**，这篇文章就会从您的博客列表中隐藏，但不会被删除。您可以随时重新 `Reopen` 它来再次发布。

### 更新文章

直接编辑对应的 Issue 内容即可。当您保存修改后，只需重新触发 GitHub Action（或在本地运行 `pnpm build:data`），您的博客就会同步最新的内容。

### 为文章分类

使用 GitHub 的 **Labels** 功能来为您的文章进行分类。

- 您可以为一个 Issue 添加一个或多个 Label。
- 每个 Label 都会成为文章的一个分类标签，显示在文章详情页，并可点击进入该分类的专属列表页面。
- 在 `/categories` 页面，会展示所有使用过的标签及其包含的文章数量。

## 高级技巧

### 自定义文章摘要

默认情况下，系统会自动截取文章正文的前 200 个字符作为摘要 (excerpt)。如果您想精确控制摘要的显示内容，可以在您希望摘要结束的位置，插入一个 HTML 注释：

```markdown
这是你想在摘要中显示的所有内容。
它支持多段落、引用等等。

<!-- more -->

这是分隔符之后的内容，它们不会出现在文章列表的摘要中，只会在文章详情页显示。
```

### 组织专栏文章

如果您有一系列主题相关的文章，可以将它们组织成一个“专栏”。系统会自动识别标题中包含特定模式的文章，并将它们归类到同一个专栏下。

**目前支持的专栏识别模式：**

- **标签模式 (推荐)**: 为所有属于同一个专栏的 Issues 添加一个格式为 `专栏/系列名称` 或 `column:系列名称` 的 Label。例如 `专栏/React源码深度解析`。这是最精确、最推荐的方式。

<Callout type="info">
  #### 本地算法补充
  如果未使用标签模式，我们的脚本会有一个基于标题的**兜底算法**来尝试识别专栏，但精确度有限。它会寻找标题中足够长的公共前缀，例如：
  - **系列模式**: `React源码解析（一）` 和 `React源码解析（二）` - **"之"模式**:
  `深入JavaScript之this` 和 `深入JavaScript之原型链` - **连字符模式**:
  `Vue学习笔记 - 响应式原理` 和 `Vue学习笔记 - 组件通信`
</Callout>

### "关于我" 页面

要创建或更新您的“关于我”页面，非常简单：

1.  在您的 GitHub 内容仓库的**根目录**下，创建一个名为 `about.md` 的文件。
2.  使用 Markdown 语法在其中写入您的个人介绍。
3.  提交这个文件。

下一次构建时，脚本会自动拉取 `about.md` 的内容，并将其渲染到博客的 `/about` 页面。如果这个文件不存在，导航栏中的“关于我”链接会自动隐藏。
