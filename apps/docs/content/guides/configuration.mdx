# 指南：配置博客

您可以通过配置环境变量，轻松地对博客的标题、作者、页脚等信息进行个性化定制。

## 配置方式

根据您的使用场景，有两种配置方式：

1.  **GitHub Action (推荐)**: 在您的 `.github/workflows/deploy.yml` 文件中，通过 `with` 关键字传入参数。这是自动化部署时的配置方式。
2.  **本地开发**: 在项目根目录下创建一个 `.env` 文件，并写入环境变量。

<Callout type="info">
  所有可用的配置项及其对应的环境变量名称，请查阅
  [配置选项参考](/reference/environment-variables) 页面。
</Callout>

## 基础配置示例

以下是一些最常用的配置项，可以帮助您快速打造个性化的博客。

### 1. 博客标题

这是显示在浏览器标签页和网站 Header 上的主要标题。

- **环境变量**: `NEXT_PUBLIC_BLOG_TITLE`
- **Action `with`**: `NEXT_PUBLIC_BLOG_TITLE`
- **示例**:
  ```yaml
  # in deploy.yml
  with:
    NEXT_PUBLIC_BLOG_TITLE: '小羊的数字花园'
  ```
  ```env
  # in .env
  NEXT_PUBLIC_BLOG_TITLE="小羊的数字花园"
  ```

### 2. 作者名称

用于在文章元数据（Metadata）中显示作者。

- **环境变量**: `NEXT_PUBLIC_BLOG_AUTHOR`
- **Action `with`**: `NEXT_PUBLIC_BLOG_AUTHOR`
- **默认值**: 如果不设置，会自动使用您的 GitHub 用户名。
- **示例**:
  ```yaml
  with:
    NEXT_PUBLIC_BLOG_AUTHOR: 'Young Liu'
  ```

### 3. 页脚文本

自定义显示在页面底部的版权或Slogan信息。

- **环境变量**: `NEXT_PUBLIC_FOOTER_TEXT`
- **Action `with`**: `NEXT_PUBLIC_FOOTER_TEXT`
- **示例**:
  ```yaml
  with:
    NEXT_PUBLIC_FOOTER_TEXT: '© 2024 YLiu | All Rights Reserved'
  ```

### 4. 站点 URL

配置您博客的最终访问地址，这对于生成正确的 SEO `sitemap` 和 `RSS` 至关重要。

- **环境变量**: `NEXT_PUBLIC_SITE_URL`
- **Action `with`**: `NEXT_PUBLIC_SITE_URL`
- **默认值**: 如果不设置，会尝试从 GitHub Pages 的部署 URL 自动推断。
- **示例**:
  ```yaml
  with:
    NEXT_PUBLIC_SITE_URL: 'https://your-custom-domain.com'
  ```
